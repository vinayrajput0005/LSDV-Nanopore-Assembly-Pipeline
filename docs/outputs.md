# Output Documentation

## Overview

The LSDV Nanopore Assembly Pipeline generates a structured collection of analytical outputs designed to support genome reconstruction, assembly validation, quality assessment, and downstream comparative genomics analyses. Outputs are organized at both the project level and sample level to facilitate reproducible analysis, long-term data management, and integration into genomic surveillance workflows.

Each processed sample receives an independent analysis directory containing intermediate files, quality control reports, assembly products, alignment files, and coverage statistics. In addition, a project-wide summary file is generated to provide a consolidated overview of sequencing and assembly performance across all samples.

---

# Project-Level Outputs

## Pipeline_Statistics.tsv

The primary project-wide output generated by the workflow is a tab-delimited summary file containing analytical metrics for all processed samples.

```text id="juhvh5"
Pipeline_Statistics.tsv
```

This file serves as a consolidated report of sequencing, host depletion, assembly, and coverage statistics and is intended for rapid assessment of workflow performance across multiple samples.

### Reported Metrics

| Metric           | Description                                                         |
| ---------------- | ------------------------------------------------------------------- |
| Sample           | Unique sample identifier                                            |
| RawReads         | Total number of sequencing reads prior to filtering                 |
| HumanRemoved     | Reads removed following alignment to the human reference genome     |
| VeroRemoved      | Reads removed following alignment to the Vero cell reference genome |
| RSVRemoved       | Reads removed following alignment to the RSV reference genome       |
| RemainingReads   | Reads retained for assembly after all depletion stages              |
| RemainingPct     | Percentage of original reads retained for assembly                  |
| AssemblySize(bp) | Length of the final scaffolded genome assembly                      |
| Contigs          | Number of assembled contigs                                         |
| N50(bp)          | Assembly continuity statistic reported by QUAST                     |
| MeanCoverage     | Mean sequencing depth across the final assembly                     |

This file is particularly useful for comparative analysis of sequencing runs, monitoring assembly success rates, and identifying samples requiring further investigation.

# Sample-Level Outputs

Each processed sample is assigned an independent analysis directory.

Example:

```text id="1dhx6g"
Sample_01/
```

The contents of this directory represent the complete analytical record for a given sample.

---

# Quality Assessment Outputs

## NanoPlot Reports

Directory:

```text id="mjlwmv"
nanoplot/
```

This directory contains graphical and tabular quality control summaries generated directly from the raw sequencing data.

Reported metrics typically include:

* Total sequencing yield
* Read length distribution
* Read quality score distribution
* Read length N50
* Cumulative yield curves
* Read count statistics

These reports provide an initial assessment of sequencing performance and facilitate identification of potential sequencing or library preparation issues.

The NanoPlot output is intended for quality evaluation and does not directly influence downstream assembly steps.

---

# Host Depletion Outputs

The host depletion stage generates alignment files corresponding to each depletion reference.

---

## Human Alignment File

```text id="1g1b1w"
human.bam
```

Contains alignments of sequencing reads against the human reference genome.

This file represents reads identified as human-derived and removed from subsequent analysis.

---

## Vero Cell Alignment File

```text id="i3b1j4"
vero.bam
```

Contains alignments of sequencing reads against the *Chlorocebus sabaeus* reference genome.

These reads are removed to eliminate contamination originating from cell-culture systems.

---

## RSV Alignment File

```text id="hqr96f"
rsv.bam
```

Contains alignments of sequencing reads against the Respiratory Syncytial Virus reference genome.

These alignments are retained primarily for auditability and contamination tracking.

---

# Filtered Read Dataset

## Non-Host Read Set

```text id="2krrs7"
sample.non_host.fastq.gz
```

This file contains the final set of sequencing reads retained after all depletion stages.

The dataset represents the input used for de novo genome assembly and is expected to contain predominantly LSDV-derived sequences.

For most analyses, this file constitutes the primary sequencing dataset of interest.

---

# Assembly Outputs

## Flye Assembly Directory

```text id="0uh3ih"
flye/
```

This directory contains the outputs generated during de novo genome assembly.

### Primary Assembly

```text id="0jok8u"
flye/assembly.fasta
```

This file contains the draft genome assembly reconstructed directly from the filtered Nanopore sequencing reads.

The assembly serves as the initial representation of the viral genome prior to polishing and scaffolding.

Because Nanopore sequencing technologies exhibit characteristic sequencing errors, this assembly should be considered an intermediate product rather than the final consensus genome.

---

# Consensus Polishing Outputs

## Medaka Directory

```text id="d56n7j"
medaka/
```

Contains outputs generated during neural network-based consensus polishing.

### Polished Consensus Assembly

```text id="jwmbqn"
medaka/consensus.fasta
```

This file represents the error-corrected assembly generated after Medaka polishing.

The polished consensus generally exhibits improved nucleotide accuracy relative to the draft Flye assembly and serves as the input for reference-guided scaffolding.

---

# Final Genome Assembly

## RagTag Directory

```text id="krzw5h"
ragtag/
```

Contains outputs generated during reference-guided scaffolding.

### Final Scaffolded Assembly

```text id="f6l6k5"
ragtag/ragtag.scaffold.fasta
```

This file represents the primary deliverable generated by the pipeline.

The scaffolded assembly incorporates information from de novo assembly, consensus polishing, and reference-guided ordering to produce the most complete genome reconstruction possible from the available sequencing data.

For downstream analyses including phylogenetics, comparative genomics, annotation, and variant discovery, this assembly should be considered the definitive genome sequence generated by the workflow.

---

# Alignment and Mapping Outputs

## Final Read Alignment

```text id="6xjggl"
final.bam
```

Contains alignments of filtered sequencing reads against the final scaffolded assembly.

This file serves as the basis for coverage calculations, assembly validation, and manual inspection of genomic regions.

---

## BAM Index

```text id="lvrbdv"
final.bam.bai
```

Index file associated with the final BAM alignment.

This file enables rapid random access to specific genomic coordinates and is required for genome browser visualization.

---

# Coverage Analysis Outputs

## Per-Base Coverage Profile

```text id="qax6g0"
per_base_coverage.depth.txt
```

This file contains nucleotide-resolution sequencing depth information generated using SAMtools.

Structure:

```text id="2nkwhw"
Reference    Position    Depth
```

Example:

```text id="z9wqcl"
LSDV    1      425
LSDV    2      428
LSDV    3      431
```

Coverage profiles are useful for:

* Identifying low-coverage regions
* Detecting sequencing dropouts
* Assessing assembly support
* Evaluating genome completeness
* Investigating potential structural anomalies

The mean depth reported in the summary statistics is derived from this file.

---

# Assembly Quality Assessment

## QUAST Directory

```text id="4yftnh"
quast/
```

Contains assembly quality metrics generated by QUAST.

Important outputs include:

```text id="ajmvnz"
report.tsv
report.html
```

Metrics reported by QUAST include:

* Total assembly size
* Number of contigs
* N50
* GC content
* Reference alignment statistics
* Structural assembly metrics

These outputs provide standardized assembly quality measurements and facilitate comparison between assemblies generated from different samples or sequencing runs.

---

# Summary Report

## Sample Summary File

```text id="e6bx5u"
summary.txt
```

The summary report provides a concise overview of all major analytical metrics generated during processing.

Reported values include:

* Raw sequencing reads
* Host depletion statistics
* Retained read counts
* Assembly size
* Number of contigs
* N50
* Mean genome coverage
* Final assembly location

This file is intended for rapid review and provides a convenient summary of overall sample performance without requiring inspection of individual analytical outputs.

---

# Recommended Downstream Applications

The final scaffolded genome assembly and associated coverage data may be used for:

* Genome annotation
* Comparative genomics
* Phylogenetic reconstruction
* Molecular epidemiology
* Variant discovery
* Transmission analysis
* Genomic surveillance
* Reference database generation

For most downstream analyses, the primary files of interest are:

```text id="z4r8t8"
ragtag/ragtag.scaffold.fasta
per_base_coverage.depth.txt
final.bam
Pipeline_Statistics.tsv
```

Together, these outputs provide a comprehensive representation of genome structure, sequencing support, assembly quality, and analytical performance.
